GIT/Git Flow Task

1) 	Кратко опишите что делает каждая команда в GIT
git init - команда для создания локального репозитория.
           Будет создана папка .git, содержащая основу GIT-репозитория
git clone - клонирование удаленного репозитория в локальный
git add - файлы добавляются в состояние staged, делаются подготовленными к commit
git commit - добавление нового коммита
git push - внести изменения на удаленный репозиторий
git pull - забрать изменения из удаленного репозитория и применить в нашем локальном репозитории
git branch - создание новой ветки
git checkout - переключение на какую-либо ветку
git merge - слияние веток
git rebase - перемещение. Можно взять все изменения, которые попали в коммиты на одной из веток,
            и повторить их на другой
git cherry-pick - берёт изменения, вносимые одним коммитом, и пытается повторно применить их в виде нового коммита в текущей ветке.
git status - показывает состояние всех файлов в репозитории
git log - показывает историю коммитов данной ветки
git reset - переключает HEAD на последний коммит и все изменения, добавленные после него, будут доступны как
            untracked в рабочем директории.
git revert - полная противоположность git cherry-pick. Она создаёт новый коммит, который вносит изменения,
             противоположные указанному коммиту, по существу отменяя его.
git stash - берет изменённое состояние рабочего каталога, то есть изменённые отслеживаемые файлы и проиндексированные изменения,
            и сохраняет их в хранилище незавершённых изменений, которые можно в любое время применить обратно.
git diff - показывает, что вы изменили, но еще не проиндексировали, и что вы проиндексировали и собираетесь
           фиксировать; показывает непосредственно добавленные и удаленные строки
